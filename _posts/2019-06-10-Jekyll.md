###  jekyll on CentOS7

## 阿里 yum 源：

* 备份

```shell
cp /etc/yum.repos.d/CentOS-Base.repo /etc/yum.repos.d/CentOS-Base.repo.backup
```

* 添加阿里源

```shell
wget -O /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo

# 或者

curl -o /etc/yum.repos.d/CentOS-Base.repo http://mirrors.aliyun.com/repo/Centos-7.repo

```
* update

```shell
yum  clean all
yum  makecache
yum -y update
```
#### install jekyll

* 安装ruby
```shell
yum install ruby

ruby -v

gem -v
```
* 安装新版本ruby
```shell
gpg --keyserver hkp://keys.gnupg.net --recv-keys 409B6B1796C275462A1703113804BB82D39DC0E3 7D2BAF1CF37B13E2069D6956105BD0E739499BDB

 \curl -sSL https://get.rvm.io | bash -s stable

source /etc/profile.d/rvm.sh

rvm install ruby

ruby -v

```

* install jekyll

```shell
gem install jekyll

jekyll -v

gem install bundler
```

#### 创建 jekyll-blog
* 建立blog 项目
```shell
cd /opt/
jekyll new blog

New jekyll site installed in /opt/blog
```
* 预览本地blog项目信息
```shell
cd blog/
bundle exec jekyll serve
```
* 公开jekyll-blog 项目到网络

```shell
jekyll serve --host IP_Address 
# IP_Address 为本地URL
```

#### jiekyll 目录结构
```
├── _config.yml 
├── _data
|   └── members.yml
├── _drafts
|   ├── begin-with-the-crazy-ideas.md
|   └── on-simplicity-in-technology.md
├── _includes 
|   ├── footer.html
|   └── header.html
├── _layouts
|   ├── default.html
|   └── post.html
├── _posts
|   ├── 2007-10-29-why-every-programmer-should-play-nethack.md
|   └── 2009-04-26-barcamp-boston-4-roundup.md
├── _sass
|   ├── _base.scss
|   └── _layout.scss
├── _site
├── .jekyll-metadata
└── index.html # can also be an 'index.md' with valid front matter
```
|---|---|
|---|---|
|\_config.yml |配置数据,配置写在这儿,不用记住命令|
|\_drafts |	未发布的文章|
|\_includes | 布局,插件;用{%include file.ext%}来将_/includes/file.ext引用|
|\_layouts |  l模板页面布局|
|\_posts|  发布的文章，文件格式必须符合：`YEAR-MONTH-DAY-TITLE.MARKUP`。|
|\_data | 格式化好的网站数据应放在这里。jekyll 的引擎会自动加载在该目录下所有的 yaml 文件（后缀是 .yml, .yaml, .json 或者 .csv ）。这些文件可以经由 ｀site.data｀ 访问。如果有一个 members.yml 文件在该目录下，你就可以通过 site.data.members 获取该文件的内容。|
|\_site |放置生成的页面，将它加入到你的 .gitignore 文件可能是一个好注意|
|.jekyll-metadata|该文件帮助 Jekyll 跟踪哪些文件从上次建立站点开始到现在没有被修改，哪些文件需要在下一次站点建立时重新生成。该文件不会被包含在生成的站点中，将它加入到你的 .gitignore 文件可能是一个好注意|
|index.html |如果某些文件中包含 YAML 头信息 部分，Jekyll 就会自动将它们进行转换。包括 .html, .markdown, .md, 或者 .textile 等在你的站点根目录下或者不是以上提到的目录中的文件也会被转换|
|Other Files/Folders |未被提及的目录和文件如css还有images文件夹,favicon.icon等文件都将被完全拷贝到生成的site中|
